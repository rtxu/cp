| input  | 地平线  )  )   )   (   )   (   )

注：输入序列下标从 1 开始算，因为输入序列不包含地平线


设序列起点为 b, 终点为 e，合法序列需要满足两点性质：
1. h[b] == h[e] 高度相同，也就是说，两点之间出现的 +1 和 -1 数目相同，这点很容易想到
2. h[i] >= h[b], b < i <= e，即 (b:e] 中出现的所有高度需要 >= 两端高度
   以 b = 2, e = 4 距离，h[3] = -3 < h[2]，序列为 )(，虽然出现的 +1 和 -1 数目相同，但是出现顺序却不合法

根据上面两点性质，为了最大化结果 e-b，我们需要记录每个高度的最左 **合法** 端点，当遇到相同高度的右端点时，尝试更新最终答案

如何维护每个高度的最左合法端点呢?
根据性质2，如果 h[i] 的右边出现了比 h[i] 低的高度，则 h[i] 不合法，再考虑到 **相邻两点之间的高度差为1**
初始时，left[0] = 0，合法，第一个输入元素为 )，-1 低于 0，则 left[0] 不合法，left[-1] = 1 合法，以此类推
